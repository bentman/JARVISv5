## Task T11.1 — Backend API + Controller Telemetry Surfaces
**Why**
- Resolves core backend data gaps currently blocking accurate UI behavior and diagnostics.
- Unlocks downstream frontend fixes without schema ambiguity.

**Notes / constraints (only if needed)**
- This task must complete before frontend rendering tasks that depend on backend telemetry fields.
- Keep schema changes minimal and additive to avoid breaking existing API consumers and tests.

### Sub-Task T11.1.1 — Budget API monthly projection wiring
**Why**
- Tech summary: FastAPI endpoint serialization in `backend/api/main.py` controls how budget data is projected to the UI. The monthly summary helper already exists in `backend/search/budget.py` and should be wired, not reimplemented.

**Deliverables**
- Update `/budget` response assembly to populate `monthly` using existing rolling-30-day helper.
- Preserve current daily budget response shape while adding non-null monthly projection.
- Ensure API schema compatibility with `BudgetResponse` in existing backend contract tests.

**Acceptance**
- `GET /budget` returns populated `monthly.limit_usd`, `monthly.spent_usd`, `monthly.remaining_usd`.
- `daily` and `monthly` values are internally consistent (remaining = max(limit - spent, 0)).
- Existing budget endpoint behavior remains deterministic for zero-spend cases.

**Evidence to provide (required)**
- `backend/.venv/Scripts/python -m pytest tests/unit/test_api_budget.py -q`
  - PASS excerpt: `... passed ...`
